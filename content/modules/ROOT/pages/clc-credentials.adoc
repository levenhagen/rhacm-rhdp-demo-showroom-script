= Creating credentials
:navtitle: Creating credentials

In order to create a new cluster, we will need to establish credentials first. In this part of the demo, we will create credentials to establish connection with an AWS cloud environment.

The AWS credentials are provided by the RHDP system.


== Creating the AWS credential 

Access the RHDP platform, in your service page, and under the Details tab you will see the  AWS credentials: Access Key ID, Secret Access Key, and the Base DNS/Route 53 Domain.  

The needed attributes are as follows:

[cols="1,3",options="header"]
|===
|Attribute |Value

|`Base_Domain`
|Get it from RHDP system

|`ACCESS_KEY`
|Get it from RHDP system

|`SECRET_KEY`
|Get it from RHDP system

|===

Access the **Credentials** menu. Click on Add credential -> Amazon Web Services -> Amazon Web Services (Twice).

image::clc-add-credential.png[Add Credential]

Follow the steps, and fill in the required information:

* *Name*: `aws`
* *Namespace*: default 
* *Base DNS Domain*: from RHDP
* *ACCESS KEY ID*:  from RHDP
* *SECRET ACCESS KEY ID*:  from RHDP
* *Skip Proxy*
* *Pull-secret:* follow the steps in the console, or your facilitator you help you get one.
* *SSH Keys*: Use an existing key pair or generate a new one (https://docs.redhat.com/de/documentation/openshift_container_platform/4.22/html/installing_on_bare_metal/user-provisioned-infrastructure#ssh-agent-using_installing-bare-metal[see link]). Note that the key pair needs to be in the PEM format.
* *Review* and finish.


In the end, you should see something like this:

image::clc-credentials.png[AWS Credentials]

== Establishing connection with AWS via bastion/terminal

In the bastion machine, we will need to run `aws configure` to establish programatic access to AWS.

[source,bash,subs="attributes"]
----
 ssh {bastion_ssh_user_name}@{bastion_public_hostname}
----

And, then:
[source,bash]
----
 aws configure
----

[source,bash]
----
AWS Access Key ID [None]: <<your AWS Access Key ID>>
AWS Secret Access Key [None]: <<your AWS Secret Access Key ID>>
Default region name [None]: us-east-2
Default output format [None]: <<Press Enter>>
----
